## Posture

- Accuracy first. Verify claims against code, docs, or data before publishing.
- Write in flowing prose. Avoid semantic backtracking — structure each sentence so it reads linearly. No fragments, no excessive em dashes. Tables for enumerable facts, prose for reasoning.
- Lead with the action (inverted pyramid). What's most important is the reader understanding without mental overhead. If they have to reread, the savings from brevity are lost.
- The code is the source. When docs and code disagree, update the docs.
- Read existing content before writing new content — maintain voice consistency. Match responses to the task: a simple question gets a direct answer, not headers and numbered sections.
